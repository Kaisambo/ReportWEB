# Форма для аналитического отчёта по звонкам

Кастомная форма на GitHub Pages, которая отправляет запрос в воркфлоу n8n. Вместо нативной формы n8n (`/form/...`) — отдельная страница с фирменным дизайном и хостингом на GitHub.

## Архитектура

```
[user] ──► yourname.github.io/calls-form/
              │
              │ POST multipart/form-data
              ▼
       n8n Webhook (n8n.orbitai.ru/webhook/...)
              │
              ├──► Respond: Accepted (моментальный ответ странице)
              ▼
       основной воркфлоу (1-15 минут в фоне)
              │
              ▼
       отчёт в Telegram (на @username из формы)
```

Страница не ждёт окончания обработки — она получает «Принято в работу» за секунду и закрывает соединение. Готовый отчёт прилетает в Telegram.

## Что в пакете

| Файл | Что делает |
|---|---|
| `index.html` | разметка формы |
| `style.css` | стили (Inter, цвета `#0f3460` / `#0088cc`, та же палитра, что в HTML-отчёте) |
| `app.js` | валидация, отправка fetch + multipart, обработка ответа |
| `config.js` | URL вебхука и username TG-бота — **сюда подставляете свои значения** |

## Шаг 1 — настроить воркфлоу в n8n

1. **Импортируйте** `Аналитический_отчёт_по_звонкам_v3.23.json` (рядом с этим пакетом).
2. Откройте ноду **`Webhook (form POST)`**:
   - Метод: `POST`
   - Path: какое-то значение (например, `transcripts-v1`)
   - Скопируйте **Production URL** (правый клик на ноде → Copy → Production URL).
   - Должен быть вида `https://n8n.orbitai.ru/webhook/transcripts-v1`.
3. Активируйте воркфлоу (`Active: ON`). Без активации webhook не работает.
4. Убедитесь, что воркфлоу `TG Bot Registration` тоже активен (для регистрации username → chat_id).

## Шаг 2 — настроить config.js

Откройте `config.js`, подмените два значения:

```js
window.APP_CONFIG = {
  WEBHOOK_URL: 'https://n8n.orbitai.ru/webhook/transcripts-v1',  // <- ваш Production URL
  TG_BOT_USERNAME: 'a101_reports_bot'                            // <- username бота без @
};
```

## Шаг 3 — задеплоить на GitHub Pages

1. Создайте новый публичный репозиторий, например `calls-form`.
2. Загрузите туда четыре файла: `index.html`, `style.css`, `app.js`, `config.js`.
3. В репозитории: **Settings → Pages**:
   - Source: **Deploy from a branch**
   - Branch: **main** / **/(root)** → Save
4. Через 1-2 минуты страница появится по адресу:
   `https://YOUR_USERNAME.github.io/calls-form/`

## Шаг 4 — проверить

Откройте страницу в браузере, заполните форму с режимом «Из архива», нажмите Сформировать. Должно прийти зелёное сообщение «Запрос принят в работу». В Telegram через несколько минут — отчёт.

Если красное «Не удалось отправить» — см. раздел «Если что-то не работает».

## Особенности

### CORS

n8n Webhook node по умолчанию отвечает с заголовком `Access-Control-Allow-Origin`, потому что в JSON указано `allowedOrigins: '*'`. Это разрешает запрос с любого домена — в том числе с `*.github.io`.

Если хочется ограничить только своим доменом — в ноде `Webhook (form POST)`:
- Options → Allowed Origins → `https://YOUR_USERNAME.github.io`

### Загрузка файла

В режиме «Загрузить файл» страница отправляет `multipart/form-data` — n8n получает файл в `$binary.transcript`. Воркфлоу адаптирован: `Extract File Text` ищет файл сначала под этим именем, потом fallback на `binary.file` (для совместимости со старой формой).

Лимит на размер файла — настраивается в n8n переменной окружения `N8N_PAYLOAD_SIZE_MAX` (по умолчанию 16 МБ). Для текстовых транскриптов хватит с большим запасом.

### Telegram

Если username не указан — отчёт не уйдёт в TG, но воркфлоу всё равно отработает. Чтобы получить результат, вернитесь на страницу с указанным username и запустите ещё раз.

## Если что-то не работает

**На странице красное «Не удалось отправить (Failed to fetch)»** —
- Проверьте, что в `config.js` URL вебхука правильный, начинается с `https://`
- Воркфлоу в n8n активен (Active: ON)
- В консоли браузера (F12 → Console) видите CORS-ошибку? → в ноде Webhook добавьте Allowed Origins.

**TG-кнопка ведёт никуда** —
- Подставлен ли `TG_BOT_USERNAME` в `config.js` (без @, только сам username бота).

**Запрос ушёл, ответ зелёный, но отчёт не приходит** —
- В n8n откройте Executions, посмотрите последний — на каком шаге упало.
- Скорее всего OpenAI не смог обработать чанк, или Sheet вернул пусто. Логика та же, что для штатной формы воркфлоу.

**`Webhook (form POST)` не получает данные** —
- Production URL отличается от Test URL. Production срабатывает только когда воркфлоу `Active: ON`. В тестовом режиме (Test URL) нужно нажимать «Listen for test event» каждый раз перед отправкой.

## Что можно улучшить дальше

- **Polling статуса**: добавить второй вебхук `/status/<id>`, страница опрашивает его каждые 5 секунд и показывает прогресс. Сейчас просто «Ждите в Telegram».
- **Список сотрудников из Sheet**: вместо текстового поля «Сотрудник» подгружать dropdown через ещё один webhook, который читает уникальные операторы из Sheet. Для версии 1.0 не критично.
- **Preview в форме**: показать, сколько звонков будет в анализе при выбранных фильтрах, до отправки. Тоже через дополнительный webhook.
